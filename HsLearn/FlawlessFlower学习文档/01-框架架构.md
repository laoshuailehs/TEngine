# FlawlessFlower 框架架构详解

---

## 一、MVC 架构

项目采用经典的 MVC 模式，三个核心类都是单例：

```
Model (数据)  ←──  Controller (逻辑)  ──→  View (表现)
    ↑                ↑    ↑                   ↑
    │                │    │                   │
 游戏数据        SBox回调  玩家输入        UI/动画/音效
```

### Model (Model.cs)
- 纯数据类，存储所有游戏状态
- 包含：当前局数、轮数、倒计时、路单记录、错误码、爆机列表等
- 也包含工具方法：`GetPokerData()`（解析花色点数）、`GetRewardResult()`（解析结果）
- 包含 `WaitForSeconds` 对象池（优化 GC）

### View (View.cs)
- MonoBehaviour，挂载在场景 GameObject 上
- 管理所有 UI 引用（通过 `transform.Find()` 查找层级）
- 负责动画、音效、Canvas 切换
- 不直接修改 Model，通过 Controller 间接操作

### Controller (Controller.cs)
- 单例管理器，游戏核心逻辑中枢
- 监听所有 SBox 回调事件和 UI 事件
- 驱动 FSM 状态转换
- 协调 Model 和 View

## 二、FSM 状态机

### 状态定义 (FSMState.cs)

```csharp
public enum StateID
{
    Prepare,    // 准备阶段（等待开新局）
    Bet,        // 押注阶段（倒计时中，玩家可押注）
    Reward,     // 开奖阶段（展示结果）
    Setting,    // 设置阶段（IO 面板打开）
    Error,      // 错误阶段（异常暂停）
}

public enum Transition
{
    Prepare,    // → Prepare
    Bet,        // → Bet
    Reward,     // → Reward
    Setting,    // → Setting
    Error,      // → Error
}
```

### 状态转换图

```
                    ┌──────────┐
         游戏启动 → │ Prepare  │ ←────────────┐
                    └────┬─────┘              │
                         │ Transition.Bet     │ Transition.Prepare
                         ▼                    │ (Reset完成 / 开奖结束)
                    ┌────┴─────┐              │
                    │   Bet    │              │
                    │ (30秒)   │              │
                    └────┬─────┘              │
                         │ Transition.Reward  │
                         ▼                    │
                    ┌────┴─────┐              │
                    │  Reward  │──────────────┘
                    │ (开奖)   │
                    └────┬─────┘
                         │ Transition.Error
                         ▼
                    ┌────┴─────┐    Transition.Setting
                    │  Error   │ ←────────────────────┐
                    │ (异常)   │──── Transition.Prepare│
                    └──────────┘     (错误解决)        │
                         │                            │
                         │ Transition.Setting         │
                         ▼                            │
                    ┌──────────┐                      │
                    │ Setting  │──────────────────────┘
                    │ (设置)   │    Transition.Prepare
                    └──────────┘    (退出设置)
```

### 状态切换流程（核心循环）

```
Prepare ──(定时器)→ Bet ──(倒计时结束)→ Reward ──(动画结束)→ Prepare

详细：
1. Prepare: SBoxIdea.BattleResetGame() → 回调触发 Bet
2. Bet: CountDown → 30秒倒计时 → SBoxIdea.BetsStop() → 开牌动画 → Reward
3. Reward: 展示结果 → 等待玩家选卡牌 → SBoxIdea.BattleEndRound() → Prepare
4. Prepare: 打印（如需）→ SBoxIdea.BattleNewRound() → 新局开始
```

### 状态类结构

每个状态由一个 Manager + State 对组成：
```csharp
// 例: BetManager.cs
public class BetManager : FSMBaseManager
{
    public BetManager()
    {
        BetState state = new BetState();
        FSMSystem.Instance.AddState(state);
    }
}

// BetState.cs
public class BetState : FSMState
{
    public BetState()
    {
        stateID = StateID.Bet;
        AddTransition(Transition.Reward, StateID.Reward);  // 可从Bet→Reward
        AddTransition(Transition.Error, StateID.Error);    // 可从Bet→Error
    }
}
```

## 三、事件中心 (EventCenter)

项目使用事件驱动的松耦合架构，模块间通过 `EventCenter` 通信：

```csharp
// 添加监听
EventCenter.Instance.AddEventListener(EventHandle.REWARD, OnReward);

// 添加带参数监听
EventCenter.Instance.AddEventListener<int>(EventHandle.CHANGE_JACKPOT, OnChangeJackpot);

// 触发事件
EventCenter.Instance.EventTrigger(EventHandle.REWARD);
EventCenter.Instance.EventTrigger(EventHandle.CHANGE_JACKPOT, 1000);
```

### 事件分类

| 来源 | 事件前缀 | 示例 | 含义 |
|------|---------|------|------|
| SBox 硬件 | `SBOX_*` | `SBOX_BATTLE_GET_STATE` | 硬件状态推送 |
| 游戏内部 | 无前缀 | `REWARD`, `ACTIVE`, `ERROR` | 游戏状态变化 |
| IO 面板 | `IO_*` 或 `IOT_*` | `COINT_IN`, `TICKET_OUT` | 支付设备 |
| 网络 | 无固定前缀 | `PLAYER_DISCONNECT` | 网络事件 |
| 按键 | `HARDWARE_*` | `HARDWARE_KEY_CLICK` | 硬件按键 |

## 四、单例系统 (BaseManager)

```csharp
// BaseManager<T> 提供单例模式
public class Model : BaseManager<Model> { }

// 使用方式
Model.Instance.curGame = 100;
FSMSystem.Instance.PerformTransition(Transition.Bet);
PlayerMgr.Instance.localPlayer;
```

所有 Manager 类继承 `BaseManager<T>`，通过 `Instance` 访问。

`MonoBehaviour` 类型的单例继承 `MonoSingleton<T>`（如 `AudioManager`）。

## 五、定时器系统 (Timer)

```csharp
// 循环定时器（每0.25秒执行一次）
Timer.LoopAction(0.25f, (loopTimes) => { LoopCheck(); });

// 延迟执行（1秒后执行一次）
Timer.DelayAction(1, () => { DoSomething(); });

// 取消定时器
timer.Cancel();
// 重启定时器
timer.Restart();
```

此外 Model 中有自定义的 `WaitForSeconds` 对象池：
```csharp
// 避免频繁 new WaitForSeconds 造成 GC
yield return Model.Instance.DelayBymsPublic(1000);  // 等待1000ms
```

## 七、网络层

```
主机 (PlayerId=1)
  ├── 启动 ServerWS（WebSocket 服务端）
  └── 作为 ClientWS 连接自己

分机 (PlayerId=2~10)
  └── ClientWS 连接到主机

消息格式: MsgInfo { cmd, id, jsonData }
  序列化: Newtonsoft.Json
  传输层: WebSocket
```

### 消息方向

```csharp
// S2C (Server to Client): 主机 → 分机
S2C_Start, S2C_StopBet, S2C_Reward, S2C_CcountDown,
S2C_SetRecod, S2C_RefreshGameInfo, S2C_PlayerWinLock, ...

// C2S (Client to Server): 分机 → 主机
C2S_JackBet, ...
```

### 分机通信流程

```
分机按键 → Player.LocalPlayerChangeInState() 
    → NetMessageController 发消息给主机
        → 主机更新该玩家状态
            → 主机通过 SBoxIdea.BattlePlayerInState() 通知算法卡
```

## 八、热更新 (HybridCLR)

```
HybridCLRData/HotUpdateDlls/Android/
    ├── Base.dll    # 基础框架 DLL
    └── Game.dll    # 游戏逻辑 DLL

Assets/StreamingAssets/Lib/
    ├── Base.dll    # 随包基础 DLL
    └── Game.dll    # 随包游戏 DLL
```

游戏逻辑可以热更新：修改 C# 代码 → 编译 DLL → 替换热更目录的 DLL → 重启即生效。
