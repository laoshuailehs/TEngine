# FlawlessFlower 游戏逻辑玩法整理

---

## 一、游戏概述

**FlawlessFlower（花无缺）** 是一款多人扑克牌押注游戏。

- **玩家数**: 最多 10 人（PlayerId 1~10，1 号为主机）
- **每轮局数**: 100 局
- **押注对象**: 5 门花色（黑桃、红心、梅花、方块、皇冠）
- **结果判定**: 每局开一张扑克牌，花色匹配即中奖
- **倒计时**: 每局 30 秒押注时间（可配置）

## 三、完整游戏循环

### 一局游戏的详细流程

```
┌─────────────────────────────────────────────────────┐
│                    Prepare 状态                       │
│  1. 打印（如需）                                      │
│  2. SBoxIdea.BattleNewRound()                       │
│  3. 等待打印完成                                      │
│  4. GetGameNumber()                                 │
│  5. Reset() → BattleResetGame()                     │
└──────────────────────┬──────────────────────────────┘
                       │ Transition.Bet
┌──────────────────────▼──────────────────────────────┐
│                     Bet 状态                          │
│  1. 播放 BGM (随机 0-3)                              │
│  2. CountDownStart() → 30秒倒计时开始                │
│  3. 每秒: CountDown() → SBoxIdea.BetsCountDown(n)   │
│  4. 打开押注灯 (TurnOnBetLight)                      │
│  5. 玩家可以押注 ← 核心交互阶段                       │
│  6. 倒计时 5 秒: 播放倒计时音效                       │
│  7. 倒计时 0: SBoxIdea.BetsStop()                   │
│  8. 切换 BGM (rewardBgm)                             │
│  9. SBoxIdea.BattleLeadStart() → 请求开牌            │
│  10. 关闭押注灯                                      │
└──────────────────────┬──────────────────────────────┘
                       │ Transition.Reward
┌──────────────────────▼──────────────────────────────┐
│                    Reward 状态                        │
│                                                      │
│  根据 luckyId 分支:                                   │
│                                                      │
│  luckyId=4/5/7 (加倍/翻倍/大满贯):                   │
│    → 播放幸运座位音效                                 │
│    → 显示 LuckySeatTips                              │
│    → ShowCardList() → 三张卡牌飞入                   │
│    → BattLuckShow() → 启动10秒选牌倒计时              │
│    → 玩家选牌 或 超时自动选                           │
│    → TreasureAnimation() → 卡牌翻开动画               │
│    → BattLuckPrize() → 通知硬件开奖                   │
│    → ShowResult() → 显示扑克牌结果                    │
│                                                      │
│  luckyId=6 (明牌):                                    │
│    → ShowResultImmediately() → 直接显示                │
│    → 跳过扑克牌翻牌动画                               │
│                                                      │
│  luckyId=8 (彩金/Jackpot):                            │
│    → ShowCoinEff() → 金币特效                         │
│    → BattLuckPrize() → 通知硬件                       │
│    → DelayShowResult() → 3秒后显示结果                 │
│                                                      │
│  luckyId=0 (普通):                                    │
│    → ShowResult() → 普通扑克牌翻牌动画                 │
│                                                      │
│  所有分支最终:                                        │
│    → 动画播放 → BattleLeadStop()                      │
│    → 等待5秒 → MovePlayerScore() 结算                 │
│    → 等待4秒 → 翻牌动画结束                           │
│    → 等待2秒 → Transition.Prepare                    │
│    → 如有错误则 Transition.Error                      │
└──────────────────────┬──────────────────────────────┘
                       │ Transition.Prepare
                       ▼
              (回到 Prepare, 开始下一局)
```

## 四、幸运座位类型 (luckyId)

| luckyId | 名称 | 效果 | 触发条件 | 玩家交互 |
|---------|------|------|---------|---------|
| 0 | 无 | 普通开牌 | 无人中特殊奖 | 无 |
| 4 | 加倍 | 中奖分数加倍 | 算法卡判定 | 选1张卡牌 |
| 5 | 翻倍 | 中奖分数翻倍 | 算法卡判定 | 选1张卡牌 |
| 6 | 明牌 | 开局即知结果 | 算法卡判定 | 无（提前展示） |
| 7 | 大满贯 | 5门全中 | 算法卡判定 | 选1张卡牌 |
| 8 | 彩金 | 中 Jackpot | 算法卡判定 | 无 |

### luckyId=6（明牌）的特殊处理

```csharp
// Controller.OnGameStart() 中提前触发
if (Model.Instance.curRecord.luckyId == 6)
{
    AudioManager.Instance.PlayVoiceEff("ch_showCard");
    view.ShowResultImmediately(Model.Instance.curRecord);  // ← Bet 阶段就展示
    // 不打开押注灯（因为结果已出）
}
else
    SandboxController.Instance.TurnOnBetLight();
```

## 五、押注系统

### 玩家押注流程

```
玩家按下硬件按键 A/B/C/D/E
    → SBoxSandboxListener 检测
        → Player.LocalPlayerChangeInState(state, true)
            → 更新 inState (bit位)
            → 标记 isChangeState = true
            → 触发 PLAYER_INSTATE_CHANGE 事件
                → Controller.OnPlayerStateChange()
                    → 收集所有玩家 inState
                    → SBoxIdea.BattlePlayerInState(list) → 发送给算法卡
```

### inState 的押注相关位

```csharp
// 按下黑桃键 → inState 的 bit0 设为 1
// 松开黑桃键 → inState 的 bit0 设为 0
// 同时按多个键 → 多门同时押注

// 例: 同时押黑桃+皇冠 → inState = 0b10001 = 0x11
```

### 押注限制

1. **仅在 Bet 状态可押注** → `FSMSystem.Instance.CurrentStateID == StateID.Bet`
2. **爆机玩家不可押注** → `Model.Instance.winLockPlayers.Contains(playerId)`
3. **不在线玩家不可押注** → `Player.IsOnline`

## 六、爆机系统 (WinLock)

### 爆机触发条件

```csharp
// CheckClientWinlockWithOutStateCredit() 中
if (outState.credit > PlayerWinLock阈值)
    → 该玩家爆机

// OnGetState() 中
if (state.PlayerWinLock != 0)
    → 硬件返回的爆机状态 (bit位表示哪些玩家爆机)
```

### 爆机效果

```
对爆机玩家:
  - 不能押注（inState 被强制设为 STATE_EXIST）
  - 押注灯关闭
  - 显示爆机提示

对主机 (PlayerId=1):
  - TurnOffBetLight() 关闭所有押注灯
  - ClientWinLockTips 显示提示

对分机 (PlayerId≠1):
  - 发送 S2C_PlayerWinLock 消息
  - 分机自行处理 UI
```

## 七、路单系统 (Records)

### 数据结构

```csharp
class RecordStruct
{
    int value;      // 扑克牌编码 (高4位花色 + 低4位点数)
    int luckyId;    // 幸运座位类型 (0-8)
    int game;       // 局号 (1-100)
    int extraData;  // 额外数据 (小游戏参数)
    int luckyPlayerId; // 中奖玩家ID
}
```

### 统计

```csharp
Model.Instance.recordTypeCounts  // List<int> 长度5
// [0] 黑桃出现次数
// [1] 红心出现次数
// [2] 梅花出现次数
// [3] 方块出现次数
// [4] 皇冠出现次数
```

### 路单生命周期

```
新轮开始 → RefreshOnNewRound():
  records.Clear()
  recordTypeCounts = {0,0,0,0,0}

每局开奖 → UpdateRecord(record, gameIndex):
  records.Add(record)
  recordTypeCounts[result-1]++

初始化加载 → OnGetCompletedGame(data):
  遍历 data[1..100] → 逐个 Add

错误结束 → ErrorEnd(data):
  同上，但不清空
```

## 八、错误码系统

### 错误码速查

```csharp
// Model.AddErrorCode(errCode)
0:  按键档机 (提示"EndRound")
1:  本轮爆机
2:  全台爆机 (总输分爆机)
3:  盈余档机
4:  保单箱未连接
5:  打印机缺纸
6:  非法开箱1
7:  非法开箱2
8:  运行时间到 (时间打码相关)
9:  历史数据过大
10: 正在打印
11: 打印机切刀故障
12: 打印机门未关闭
```

### 错误处理流程

```
AddErrorCode(errCode)
    → 添加到 errorCodeList
    → 如果是关键错误 (4/5/6/7/11/12)，标记 isErrwhenprinting
    → PopTips 显示错误提示

RemoveErrorCode(errCode)
    → 从 errorCodeList 移除
    → 检查是否还有错误
    → 如果没有错误，自动添加 ErrorCode 0 (需要手动重开)
    → 触发 ACTIVE 事件，尝试恢复
```

### 打印机错误的特殊处理

```
SBoxPrinterStateCustom() 每0.25s轮询:
  返回1 → AddErrorCode(10) "正在打印"
  返回2 → AddErrorCode(5)  "缺纸", 移除10
  返回3 → AddErrorCode(4)  "保单箱未连接", 移除5/10
  返回4 → AddErrorCode(11) "切刀故障", 移除4/5/10
  返回5 → AddErrorCode(12) "门未关闭", 移除4/5/10/11
  返回0 → RemoveErrorCode(4/5/10/11/12)
```

## 九、打印系统

### 打印模式 (PrintMode)
| 值 | 含义 |
|----|------|
| 0 | 不打印 |
| 1 | 正常打印 |
| 2 | 伸缩打印 |

### 打印模式 (NewGameMode)
| 值 | 含义 |
|----|------|
| 0 | 自动打印 |
| 1 | 手动打印 (需按键确认) |

### 打印流程 (Prepare 状态)

```
StartNewRound()
    → DelayStartNewRoundCorountine()
        → [PrintMode != 0] 等待 CheckTime 秒
        → [NewGameMode=0] DelayStartNewRound2() → 自动打印
        → [NewGameMode=1] wait4Print=true → 等待手动按键

手动按键:
    硬件 SWITCH_RULE 按下
    → OnPrintBtnClick()
        → oldNewGameMode=1
        → DelayStartNewRound2()

DelayStartNewRound2():
    → 等待 PrinterStateCustom()==0
    → inPrintTime=true
    → BattleNewRound()

BattleNewRound 回调:
    → 等待 PrinterStateCustom()==0 (最多轮询)
    → inPrintTime=false
    → GetGameNumber() → Reset() → 新局开始
```

## 十、声音系统

### BGM 切换逻辑
```
Bet 状态: "normalBgm" + random(0,1,2)  // 押注阶段
Reward 状态: "rewardBgm" + random(0,1)  // 开奖阶段
CountDown=5: 播放 "countDown" 音效
中 Jackpot: 播放 "jackpot" 语音 + "coinEff" 音效
```

### 扑克牌翻牌音效
```
先播放花色名: "spade" / "heart" / "club" / "diamond"
等待1.4秒
再播放点数: "value_1" ~ "value_15"
```
