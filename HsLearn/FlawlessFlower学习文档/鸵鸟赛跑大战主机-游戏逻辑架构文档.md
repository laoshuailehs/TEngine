## 1. 项目概览

### 1.1 项目简介

这是一个基于 Unity 的**鸵鸟赛跑街机游戏**，运行在 Android 街机柜上。游戏模拟 5 只鸵鸟（游戏中称"马"）在赛道上竞赛，玩家通过分机终端下注，根据比赛结果赢取奖励。

### 1.2 核心玩法

- **5 只鸵鸟**参加每场比赛，每只有不同的奔跑类型（匀速型、后期爆发型、前期爆发型、一直领先型、一直落后型）
- 玩家在**下注阶段**从 10 个结果（对应 5 选 2 的第一二名组合）中选择下注
- 比赛结果由**外部算法硬件 SBox** 预先计算，通过 TCP 发送给主机
- 主机负责渲染比赛画面、驱动动画、播放语音解说
- 支持**4种大奖模式**：普通、射灯（送灯）、通通有奖（大满贯）、彩金

### 1.3 硬件架构

```
┌─────────────────────────────────────────────────────┐
│                   街机柜物理布局                       │
│                                                       │
│  ┌─────────┐                          ┌───────────┐  │
│  │ SBox    │ ← TCP/IP → 主机大屏幕    │ IoT 设备   │  │
│  │ 算法硬件 │              (本代码)    │ 投币/出票  │  │
│  └─────────┘              ↕ WebSocket └───────────┘  │
│                    ┌──────┼──────┐                    │
│                    分机1 分机2...分机6                  │
│                    (玩家下注终端)                       │
└─────────────────────────────────────────────────────┘
```

## 2. 技术架构总览

### 2.1 程序集架构

项目分为两个核心程序集（Assembly Definition）：

| 程序集 | 路径 | 职责 |
|--------|------|------|
| **Base** | `Assets/Scripts/Base/` | 基础框架：FSM、网络、音频、UI框架、对象池、事件系统、数据库、IoT支付、Timer |
| **Game** | `Assets/Scripts/Game/` | 游戏逻辑：Controller、View、Model、马匹、摄像机、FSM状态、UI面板、大奖特效 |

Game 程序集引用 Base，构成分层架构。

### 2.2 设计模式

| 模式 | 实现位置 | 用途 |
|------|---------|------|
| **单例模式** | `BaseManager<T>`, `MonoSingleton<T>` | 全局管理器（Controller、Model、PlayerMgr等） |
| **有限状态机（FSM）** | `FSMSystem`, `FSMState`, `Game/FSM/` | 游戏回合状态流转 |
| **MVC模式** | `Model.cs`, `View.cs`, `Controller.cs` | 游戏数据-视图-控制分离 |
| **事件驱动** | `EventCenter.cs`, `EventHandle.cs` | 模块间解耦通信 |
| **对象池** | `PoolMgr.cs`, `GameObjectPool.cs` | 减少 GC 和实例化开销 |
| **观察者模式** | `Message.cs`, `Messenger` | 网络消息分发 |

### 4.2 各状态详解

#### PrepareState（准备阶段）

**作用**: 重置比赛、获取赔率、初始化新一轮

```
进入状态 (DoBeforeEntering):
  1. 注册事件监听
  2. SBoxIdea.BattleResetGame() → 请求SBox重置回合
  
SBox 回应 OnResetGame(double[] odds):
  3. 检查是否需要激活
  4. 检查错误码列表
  5. 如果有错误 → 转入 Error 状态
  6. 如果没有错误 → Reset()
       ├→ 广播 S2C_Reset 给所有分机
       ├→ Model.ResetData() (清除上一局数据)
       └→ SBoxIdea.GetOdds() → 请求赔率

SBox 回应 OnGetOdds(int[] result):
  7. 保存赔率数据到 Model.odds[10]
  8. 向每个分机发送 S2C_GameInfo (包含赔率)
  9. FSMSystem.PerformTransition(Transition.Bet) → ★进入下注阶段★
```

#### BetState（下注阶段）

**作用**: 玩家下注、倒计时、等待结果

```
进入状态 (DoBeforeEntering):
  1. 注册事件监听
  2. SBoxIdea.BetsStart(countDown) → 通知SBox开始下注

SBox 回应 OnBetsStart(result):
  3. Controller.Instance.OnGameStart()
       ├→ 随机播放 BGM (normalBgm0/1/2)
       ├→ 广播 S2C_Start 给所有分机
       └→ View.CountDownStart() → 开始倒计时显示

倒计时过程:
  4. SBox 每秒推送 SBOX_BATS_COUNT_DOWN → UI更新倒计时
  5. 玩家通过分机下注，分机发送 C2S_SBoxPlayerInState

SBox 回应 OnBetsStop(result):
  6. 延迟1秒插入临时历史记录 InsertTempHistoryData()
  7. FSMSystem.PerformTransition(Transition.Reward) → ★进入开奖阶段★
```

#### RewardState（开奖阶段）

**作用**: 获取比赛结果、播放比赛画面、发放奖励

```
进入状态 (DoBeforeEntering):
  1. 注册事件监听
  2. SBoxIdea.OstrichGetOpenInfo() → 请求开奖信息

SBox 回应 OnOstarichOpenInfo(sBoxOstrichOpenInfo):
  3. Model.Instance.normalWinner = turlyResult (0-9 对应一二名组合)
  4. Model.Instance.curOpenInfo = sBoxOstrichOpenInfo
     └→ setter中为每个玩家处理 X2 泡泡数据
  5. 延迟3秒后 SBoxIdea.OstrichGetReward() → 请求奖励信息
     (延迟3秒是为了给算法卡时间计算彩金)

SBox 回应 OnGetReward(sBoxBattleLeadData):
  6. View.SetHosreRankArray() → 设置马的排名
  7. Model.Instance.curRewardInfo = sBoxBattleLeadData
     └→ setter中保存记录 DealWithRecordData()
  8. EventCenter.EventTrigger(REWARD) → 触发比赛画面播放
  9. Controller.PostBetLog() → 上报数据到后台
  
比赛播放完成后:
  10. EventCenter.EventTrigger(GAMERESET)
  11. FSMSystem.PerformTransition(Transition.Prepare) → ★回到准备阶段★
```

#### ErrorState（错误状态）

处理 4 种错误码：
- **错误码 1**: 运营时间到
- **错误码 2**: 盈余宕机 (WinLock)
- **错误码 3**: 历史数据过大
- **错误码 4**: 全台爆机 (MachineWinLock)

#### SettingState（设置状态）

打开 IO 设置画布，管理员可以调整：语言、音量、网络开关、Jackpot 参数、票据比例等。

## 5. 赛马比赛逻辑详解

### 5.1 赛道结构

赛道由 **路点系统（Waypoint）** 构成：

```
起点 → [路点1] → [路点2] → ... → [弯道路点] → ... → [终点]
                                                       ↓
                                              [颁奖房间 ShowRoom]
```

- 每两个大路点之间通过 `countPath = 50` 个插值小路点平滑连接
- 赛道上有**触发器区域**用于：
  - 摄像机切换（起跑、跟拍、弯道、终点等）
  - 语音播报触发
  - 速度微调触发
  - 最终排名调整

### 5.2 马匹类型系统 (HorseState)

每个马有一个 `HorseType` 和 `index`（0-4），共同决定其行为：

```csharp
// 5种马匹类型
enum HorseType {
    Average,        // 匀速型 - 速度始终在基准附近
    LateExplosive,  // 后期爆发型 - 前期慢，后期加速
    EarlyExplosive, // 前期爆发型 - 前期快，后期可能减速
    Leader,         // 一直领先型 - 保持在前列
    Later           // 一直落后型 - 保持在后面
}
```

**速度系统参数**:
- `baseSpeed = 20` — 基准速度
- `accele = 0.7f` — 加速度
- `reverseAccele = -0.7f` — 减速度
- `speed` 起始值 = 14，逐渐加速到 baseSpeed
- 弯道限制由 `clampAccle` 和 `clampReverseAccle` 控制
- 每个 index 有不同的弯道参数

**速度更新逻辑** (`UpdateNormalRun()`):
1. 计算与参照马的实时距离
2. 根据马匹类型 + 距离 + 排名 决定加减速
3. 速度在 `[25, 29]` 范围内波动（可以微调）
4. 射灯模式下速度固定为 22

### 5.3 名次确定机制

比赛结果由 SBox 预先确定，代码中通过以下机制保证结果与算法一致：

1. **排名映射表** (`FirstAndSecondPlaceResultsMappingTable`):
```
0 → (1, 2)    1 → (1, 3)    2 → (1, 4)    3 → (1, 5)    4 → (2, 3)
5 → (2, 4)    6 → (2, 5)    7 → (3, 4)    8 → (3, 5)    9 → (4, 5)
```
- SBox 返回 `turlyResult` (0-9)，映射为第一名和第二名的马号

2. **速度微调** (`TweakHorseSpeed`): 在比赛中段微调各马速度，让指定马匹跑在前面
3. **最后200m调整** (`FinalSpeedAdjustment`): 在距离终点200m时，根据预设结果强制调整排序

### 5.4 马匹动作状态

```csharp
enum SpotRunType {
    Run,           // 正常沿路径跑
    JumpFireCircle,// 冲向火圈
    Jumping,       // 正在跳火圈
    RunAway,       // 跳过后跑开
    StayPut,       // 摔倒停在原地
    SetSpurt       // 冲刺放置
}
```

动画状态包括: Run, Walk, Idle, SpurtRun, Lash, Win, Jump, FallDown, Cheer, EnterCurveRun, ExitCurveRun

### 5.5 比赛播放流程 (View.cs)

```
CountDownStart() → 倒计时UI动画 + 音效
  ↓
枪声 → DoorOpeningSound4 + gunshot 音效
  ↓
比赛开始 → 所有马开始跑 + BGM切换
  ↓
各触发点依次触发:
  ├→ 起跑摄像机
  ├→ 自由摄像机
  ├→ 主摄像机
  ├→ 弯道摄像机 (左弯)
  ├→ 语音播报 (领跑/追赶/激烈竞争等)
  ├→ TweakHorseSpeed (微调速度)
  ├→ 最终摄像机
  ├→ FinalSpeedAdjustment (最终排名调整)
  └→ WinTrigger → 终点线
  ↓
展示结果 → 大奖表现 (如果有) → 颁奖房间
  ↓
回放 (Replay) → 慢动作回放关键镜头
  ↓
返回准备房间 → 等待下一轮
```

## 6. 大奖系统

### 6.1 四种大奖类型

| 类型 | 枚举值 | 说明 |
|------|--------|------|
| **Normal** | 0 | 普通比赛，无特殊大奖 |
| **SendingLanterns（射灯/送灯）** | 1 | 聚光灯随机选获胜者，火球飞向盾牌特效 |
| **AllPrize（通通有奖/大满贯）** | 2 | 所有 10 个名次组合依次展示，连续播放 |
| **MosaicGold（彩金）** | 3 | 累积奖池，显示彩金排行榜 |

### 6.2 大奖触发逻辑

大奖类型由 SBox 算法硬件在 `curRewardInfo.subGameId` 中返回：

- `subGameId == 1` → 射灯
- `subGameId == 2` → 大满贯
- `subGameId == 3` → 彩金

### 6.3 射灯（SendingLanterns）详解

```
SpotLightPrize.OnInit()
  ├→ 初始化火球特效 (FireBallView)
  ├→ 初始化背景
  └→ 初始化高亮盾牌列表

流程:
  1. 显示射灯UI背景
  2. 聚光灯旋转扫描 (spotlightShiningInCircles 音效)
  3. 火球飞出 (FireBallView.TweenAction)
  4. 火球击中盾牌 → 盾牌高亮 (ShowHightLinghtGame)
  5. 释放马匹 (spotlightReleaseTheHorse 音效)
  6. 马匹跳火圈 (JumpFireCircle → Jumping → RunAway)
```

### 6.4 大满贯（AllPrize）详解

- 逐帧展示所有 10 个历史结果
- 连续播放 10 段比赛画面
- 大满贯专用 Spine 动画 (`pup_GrandSlam`)
- 全部结果展示完毕后进入颁奖

### 6.5 彩金（MosaicGold）详解

- 显示累积 Jackpot 金额
- 彩金排行榜 (`rankingList.cs`)
- 网络 Jackpot 跨机台共享 (`netWinJackpotInfoDic`)
- 大赢动画和烟花特效

## 7. 多人终端架构

### 7.1 主机-分机模型

```
主机 (本代码)
  ├── WebSocket Server (端口可配置)
  ├── 大屏幕渲染比赛画面
  ├── 连接 SBox 算法硬件
  ├── 连接 IoT 投币/出票设备
  └── 管理 1-6 个分机连接

分机 (玩家终端)
  ├── WebSocket Client 连接到主机
  ├── 小屏幕显示下注界面
  ├── 硬件按钮：下注键、确认键、兑票键
  └── 独立的 coin-in/coin-out 计数
```

### 7.2 网络消息协议

**S2C（主机→分机）命令**:
| 命令 | 说明 |
|------|------|
| `S2C_Start` | 游戏开始（倒计时启动） |
| `S2C_Reset` | 重置界面 |
| `S2C_GameInfo` | 发送赔率、局号、玩家分数 |
| `S2C_BetsCountDown` | 倒计时同步 |
| `S2C_Stop` | 停止下注 |
| `S2C_Reward` | 开奖结果 |
| `S2C_Error` | 错误信息 |
| `S2C_Jackpot` | Jackpot 数值更新 |
| `S2C_WinJackpot` | 网络 Jackpot 中奖公告 |

**C2S（分机→主机）命令**:
| 命令 | 说明 |
|------|------|
| `C2S_Login` | 分机登录 |
| `C2S_GetGameConfig` | 请求游戏配置 |
| `C2S_HARDWARD_KEYDOWN/UP` | 硬件按键事件 |
| `C2S_SBoxPlayerInState` | 玩家下注状态变更 |
| `C2S_SBoxCoinIn/Out` | 投币/出币通知 |

### 7.3 玩家管理 (PlayerMgr + Player)

```csharp
// Player.cs 关键字段
public class Player {
    int macId;           // 分机号 (1-6)
    ClientConnection client;  // WebSocket连接
    bool IsOnline;       // 在线状态（心跳检测）
    int inState;         // 玩家按键状态（下注/兑票/搬分）
    int coinInCount;     // 投币计数
    int coinOutCount;    // 出币计数
    GameInfo gameInfo;   // 本局赔率和泡泡数据
    List<HistoryRecordData> historyRecordDatas; // 最近5局历史
}
```

**心跳检测**: 每3秒检查一次，超过10秒无心跳则标记为离线。

## 8. 硬件通信系统（SBox + IoT）

### 8.1 SBox 算法硬件

SBox 是外部硬件控制器（IP: `192.168.3.116`），负责：
- **随机数生成**：决定比赛结果
- **赔率计算**：根据下注分布动态调整
- **游戏逻辑**：下注控制、奖励计算、彩金累积

```
GetOdds()            → 获取赔率 (10 个值)

BetsStop()           → 停止下注
OstrichGetOpenInfo() → 获取开奖信息 (turlyResult)
OstrichGetReward()   → 获取奖励数据 (subGameId + Data)
BattleGetState()     → 轮询状态
Jackpot()            → 获取 Jackpot 金额
WinLockBalance()     → 获取自锁余额
MovePlayerScore()    → 搬分操作
NeedActivated()      → 检查激活状态
```

## 10. 热更新系统（HybridCLR）

### 10.1 工作原理

HybridCLR 是 Unity IL2CPP 的热更新方案，允许在不重新构建 APK 的情况下更新 C# 代码：

```
编译时:
  Game.dll → 放入 StreamingAssets/Lib/

运行时:
  1. Main.cs 加载 AOT 元数据 (mscorlib, System, System.Core)
  2. Assembly.Load() 加载 Base.dll, Game.dll
  3. Hot update DLL 中的代码即可执行
```

### 10.2 更新 DLL 流程

1. 修改 `Assets/Scripts/Game/` 下的 C# 代码
2. Unity 中 Build，生成新的 `Game.dll`
3. 上传到 `res.cfkj88.com/RaceOstrichSolo/Client/Lib/Game.dll`
4. 同时更新 `Version.txt` 版本号
5. 设备重启后自动检测版本并下载新 DLL

### 10.3 AssetBundle 更新

同上，美术资源和预制体也能热更新：
- Bundle 源文件在 `Assets/Bundles/` 目录
- 构建工具: `Assets/Editor/BuildAssetBundle.cs`
- 所有 bundle 上传到 `res.cfkj88.com/.../AssetBundles/`
- 客户端自动比对版本后下载

## 16. 代码规范与调试技巧

### 16.1 事件系统使用

```csharp
// 注册事件（一般在 DoBeforeEntering 或 Init 中）
EventCenter.Instance.AddEventListener<T>(EventHandle.XXX, OnXXX);

// 触发事件
EventCenter.Instance.EventTrigger(EventHandle.XXX);
EventCenter.Instance.EventTrigger<T>(EventHandle.XXX, data);

// 移除事件（在 DoBeforeLeaving 或 OnDestroy 中）
EventCenter.Instance.RemoveEventListener<T>(EventHandle.XXX, OnXXX);
```

**重要**: 必须在对象销毁前移除事件监听，否则会导致内存泄漏和空引用异常。

### 16.2 Timer 使用

```csharp
// 延迟执行
Timer.DelayAction(3.0f, () => { /* 3秒后执行 */ });

// 循环执行
Timer.LoopAction(0.25f, (loopTimes) => { /* 每0.25秒执行 */ });

// 延迟帧执行
Timer.DelayFrameAction(2, () => { /* 2帧后执行 */ });
```

### 16.3 网络消息发送

```csharp
// 向所有分机广播
NetMgr.Instance.SendToAllClient(jsonString);

// 向特定分机发送
NetMgr.Instance.SendToClient(player.client, jsonString);

// 向远程服务器发送
NetMgr.Instance.SendToServer(jsonString);
```

### 16.4 单例获取

```csharp
// 继承 BaseManager<T> 的单例（非MonoBehaviour）
Model.Instance.xxx;
Controller.Instance.xxx;
PlayerMgr.Instance.xxx;
IOCanvasModel.Instance.xxx;

// 继承 MonoSingleton<T> 的单例（MonoBehaviour）
BigPrice.Instance.xxx;
```

### 16.5 常见坑点

1. **SBox 连接**: SBox 是必需的硬件，编辑器下测试很多功能会因 SBox 未连接而卡住。可以使用 `testMode` 字段绕过。
2. **AssetBundle 依赖**: 修改资源后必须重新 Build AssetBundle，否则不会生效。
3. **HybridCLR 限制**: Game.dll 中的代码会被热更新，但 Base.dll 中的修改在某些情况下可能不会热更新。
4. **事件泄漏**: 频繁进出 FSM 状态时，务必在 `DoBeforeLeaving` 中移除事件监听。
5. **线程安全**: 所有 Unity API 调用必须在主线程，使用 `Loom.QueueOnMainThread()` 处理回调。

### 16.6 性能优化点

- 对象池: `PoolMgr` 和 `GameObjectPool` 用于马匹效果、粒子等
- 音频预加载: `Load.cs` 中的 `PreloadSound()` 一次性加载所有音频
- AssetBundle 缓存: 已下载的 Bundle 缓存在 `persistentDataPath`
- SBox 轮询频率动态调整: 下注阶段 0.08s，其他阶段 0.25s
