## 第一周：入门阶段

### 第1天（4-6小时）

**上午: 项目跑起来**

- [ ] 阅读 `00-项目总览.md`（30分钟）
- [ ] 克隆项目、Unity 打开、能 Play 运行（1小时）
- [ ] 打开主场景，在 Hierarchy 窗口浏览 GameObject 层级结构
- [ ] 找到 View.cs 挂载的 GameObject，查看 Inspector

**下午: 跟一遍游戏循环**

- [ ] 阅读 `Load.cs` — 入口在做什么（30分钟）
- [ ] 阅读 `FSMState.cs` — 理解状态和转换枚举（15分钟）
- [ ] 阅读 `BetState.cs` → `RewardState.cs` → `PrepareState.cs`（1小时）
  - 重点关注每个状态的 `AddTransition()` 定义了哪些转换
- [ ] 在 `Controller.OnGameStart()` 打断点，Play 后跟一遍调用栈
- [ ] 手动改一下倒计时数字，验证修改生效

**产出**: 能用自己语言画出状态转换图

### 第2天（4-6小时）

## 学习技巧

### 1. 善用 IDE 功能

```
Ctrl+Click       → 跳转到定义
Ctrl+Shift+F     → 全局搜索 (比 Grep 更直观)
F12              → 查找所有引用
Ctrl+K, Ctrl+D   → 格式化代码
```

### 2. 理解代码的最佳方法

```
1. 先看注释/文档，建立概念
2. 打断点，跑一遍实际流程
3. 修改一个小地方，看效果
4. 画出流程图/时序图
5. 给别人讲解一遍
```

### 3. 关键断点位置

```
Controller.OnGameStart()       → 一局开始
Controller.OnReward()          → 开奖流程
Controller.OnBattleLeadStart() → 开牌
View.CountDownStart()           → 倒计时
View.ShowResult()               → 结果展示
Player.LocalPlayerChangeInState() → 押注操作
SandboxController.AddHardwareBtnEvent() → 按键注册
SBoxIdea.BattleGetStateR()      → 硬件数据回调
```

### 4. 各阶段重点关注

| 阶段 | 关注重点 | 忽略 |
|------|---------|------|
| 入门 | FSM 流程、MVC 结构 | IoT支付、热更新 |
| 基础 | 押注系统、按键事件 | 网络细节、打印机 |
| 进阶 | SBox API、开奖流程 | HybridCLR配置 |
| 熟练 | 错误处理、网络通信 | M2Mqtt底层 |
| 精通 | 全部 | 无 |
